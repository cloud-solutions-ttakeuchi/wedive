# テストシナリオ v3.0.0 (2025-12-26)

## 概要
本バージョンでは、Web版の検索クラッシュ修正、管理者承認ロジックの改善、およびアプリ版詳細ページ（スポット・生物）の刷新と相互連携機能を実装しました。これらの項目が期待通り動作することを確認します。

---

## 1. Web版: 生物図鑑検索と安定性
### 【ケース1-1】生物図鑑(Pokedex)での検索
- **操作**: `Search Terms` 入力欄に、タグ、エリア名、科名などを入力する。
- **期待結果**: 
  - 入力中および入力後に画面が真っ白にならず、正常にフィルタリングされること。 (TypeErrorの解消確認)
  - 検索条件をクリアした際、全件が表示されること。

### 【ケース1-2】管理画面での削除提案承認
- **前提**: `creature_proposals` または `point_proposals` に「削除提案」が存在すること。
- **操作**: 管理画面 (`/admin/proposals`) で削除提案に対し「承認」ボタンを押す。
- **期待結果**: 
  - 承認完了のアラートが表示されること。
  - Firestore上で対象ドキュメントの `status` が `rejected` に更新され、一般ユーザー画面から非表示になること。

---

## 2. モバイルアプリ版: スポット・生物詳細の連携
### 【ケース2-1】スポット詳細からの生物閲覧
- **前提**: 少なくとも1件以上の生物が紐付いているスポットを表示する。
- **操作**: スポット詳細ページ (`/details/spot/[id]`) を開く。
- **期待結果**: 
  - 「Confirmed Species（確認された生物）」セクションに生物カードが横並びで表示されること。
  - 生物カードをタップし、その生物の詳細ページへ遷移すること。

### 【ケース2-2】生物詳細からのスポット閲覧
- **前提**: 少なくとも1箇所目撃情報がある生物を表示する。
- **操作**: 生物詳細ページ (`/details/creature/[id]`) を開く。
- **期待結果**: 
  - 「Spotted at（目撃ポイント）」セクションにスポットカードが横並びで表示されること。
  - スポットカードをタップし、そのスポットの詳細ページへ遷移すること。

### 【ケース2-3】没入型デザインとナビゲーション
- **操作**: スポット・生物の詳細ページを交互に遷移する。
- **期待結果**: 
  - 下部のタブバーが表示されず、没入感のあるレイアウトであること。
  - 画面上部のパンくずリスト（🏠Homeなど）が正しく動作し、上位階層やホームに戻れること。
  - SafeArea（ノッチ、ホームインジケーター）にボタンが重ならず正しく配置されていること。
