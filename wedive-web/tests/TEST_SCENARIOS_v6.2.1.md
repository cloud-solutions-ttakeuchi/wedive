# テストシナリオ v6.2.1 - レビュー可視化・権限・エリア比較の修正

## 1. レビュー閲覧権限と可視化の検証
### 1.1 ログインユーザー（非管理者）による自分の「承認待ち」レビューの確認
- **前提**: ログインした非管理者ユーザーが未承認のレビューを1件持っている。
- **操作**:
    1. そのユーザーでログインする。
    2. レビューを投稿したポイントの詳細ページを開く。
- **期待結果**:
    - 自分の投稿したレビューが「承認待ち」バッジ付きでフィードの最上部に表示されること。
    - 他の承認済みレビューも表示されること。

### 1.2 ゲストユーザーによる閲覧
- **前提**: ログアウトした状態。
- **操作**: ポイント詳細ページを開く。
- **期待結果**:
    - 承認済みのレビューが正常に一覧表示されること。
    - 承認待ちのレビュー（他人のもの）が表示されないこと。
    - 「インデックスが必要です」等のFirebaseエラーがコンソールに出ていないこと。

## 2. エリア平均比較統計の検証
### 2.1 レーダーチャートとゲージの表示
- **前提**: 対象ポイントと同一エリア内に、他のポイント（レビューあり）が存在する。
- **操作**: ポイント詳細ページの「実測値」セクションを確認する。
- **期待結果**:
    - レーダーチャートに「実測（ポイント）」と「実測（エリア平均）」の2つの線が表示されること。
    - 視認性（透明度）の比較ゲージに、そのポイントの値とエリア平均の値が並んで表示されること。

### 2.2 フィルタリングの連動
- **操作**: 「春」「夏」などのシーズンフィルターを切り替える。
- **期待結果**:
    - ポイントの値とエリア平均の値が、両方とも指定されたシーズンのデータのみに基づき再計算されること。

## 3. レビュー投稿画面（AddReviewPage）の修正検証
### 3.1 Cカード指導団体とランクの連動
- **操作**:
    1. レビュー投稿画面の最終ステップ（レビュアー情報）へ進む。
    2. 「指導団体」を PADI から NAUI や SSI に変更する。
- **期待結果**:
    - 指導団体を変更した際、右側の「認定ランク」の選択肢が即座に切り替わること。
    - ランクが空欄にならず、新しい団体の最初のランクが初期選択されること。

## 4. パフォーマンスと安定性の検証
### 4.1 CIビルドとカスケードレンダリング
- **確認**: GitHub Actions のビルドパイプライン。
- **期待結果**:
    - `react-hooks/set-state-in-effect` エラーが発生せず、ビルドが正常に完了（Success）すること。
    - ログアウト時に無限ロードや画面のチラつきが発生しないこと。
